# Offline Image Converter

Convert and compress images entirely in your browser. Files never leave your device.

## What it does

- Converts between 7 input formats: JPEG, PNG, WebP, AVIF, HEIC, GIF, BMP
- Outputs to 5 formats: JPEG, PNG, WebP, AVIF, GIF
- Full 35-pair conversion matrix tested — see [MATRIX.md](MATRIX.md)
- Client-side quality, resize, and format controls per file
- Batch queue with pause/resume and ZIP download
- Zero uploads — no server, no telemetry

## Privacy promise

No network requests are made during conversions. After the initial page load (which fetches the app shell and lazy codec chunks from the same origin), all processing runs locally via Canvas API, WebAssembly, and Web Workers. The privacy test at `tests/e2e/privacy.spec.ts` enforces this automatically on every CI run.

## Run locally

### Dev server (recommended during development)

```bash
npm install
npm run dev
```

Opens at `http://localhost:5280` (or `http://192.168.1.93:5280` on LAN). Hot module replacement is enabled.

### Production preview via Vite

```bash
npm run build && npm run preview
```

Serves the `dist/` build at `http://localhost:5281`.

### Production preview via Apache

Build first:

```bash
npm run build
```

Then install the included vhost config:

```bash
sudo cp apache-vhost.conf /etc/apache2/sites-available/converter.conf
sudo a2ensite converter.conf
sudo apache2ctl configtest && sudo systemctl reload apache2
```

Access at `http://localhost:8081` or `http://192.168.1.93:8081`. See `apache-vhost.conf` for full details.

## Tests

```bash
# TypeScript type check
npm run typecheck

# Vitest unit tests
npm test

# Playwright E2E tests (requires dev server or auto-starts one)
npm run test:e2e

# Regenerate MATRIX.md from a full 35-cell run
npm run test:matrix
```

Playwright runs against Chromium by default. The E2E suite includes the privacy verification test that asserts zero external network requests during conversion.

## Conversion matrix

[MATRIX.md](MATRIX.md) is the source of truth. It is regenerated by `npm run test:matrix` and shows pass/fail for every input→output format pair.

## Deployment

The app is a fully static build (`dist/`). Deploy the contents of `dist/` to any static host.

**Vercel / Netlify / Cloudflare Pages / GitHub Pages**

If deployed to a sub-path (e.g. `/converter/`), set the `VITE_BASE` environment variable at build time:

```bash
VITE_BASE=/converter/ npm run build
```

This is already wired in `vite.config.ts` — no code changes required. For GitHub Pages under a `/converter/` sub-path:

```bash
VITE_BASE=/converter/ npm run build
```

For root-path deploys (`example.com/`) no variable is needed; the default `./` base works correctly.

## Tech stack

Vite + vanilla TypeScript + Vitest + Playwright. Plain CSS, no UI framework. Codec dependencies: `@jsquash/avif`, `heic2any`, `upng-js`, `gif.js`, `jszip`.

## Known limitations

- **EXIF metadata** is not preserved. The canvas re-encode path always strips metadata; no EXIF read/write library is included.
- **AVIF encode runs on the main thread.** Web Worker integration is documented but deferred — see `src/lib/workers/converter.worker.ts` for the reason. For typical image sizes this is non-blocking.
- **Animated GIF input:** only the first frame is decoded. Multi-frame GIF encode is supported for output but round-tripping animated GIFs is not.
- **Secure context requirement:** `crypto.randomUUID` is only available over HTTPS or `localhost`. Non-HTTPS LAN deployments fall back to a `getRandomValues`-based UUID generator (already handled in the codebase).
